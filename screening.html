AI财报筛选中心

把候选发现和正式公告核验拆开处理：先收集可能符合条件的公司进入候选池，再按批次核验正式公告。只有 verified 且来源通过白名单的数据才能写入正式数据库。

候选发现 正式核验 候选池进度 数据管理与本地回退

1. 设置候选发现任务

生成给联网 AI 的候选发现提示词。候选线索可以来自业绩快报、预告或财经线索，但不会直接进入正式数据库。

排除规则

2. 复制候选发现提问

要求 AI 只返回候选 JSON。候选可以带线索来源，但必须标记 needsOfficialVerification=true。

3. 粘贴候选 JSON 并加入候选池

页面会校验候选代码、报告期、财务线索和来源链接。通过的候选进入候选池，等待正式公告核验。

    1. 选择正式核验批次

    从候选池自动选择 5—8 家待核验公司。正式核验必须回到巨潮资讯或交易所公告。

    2. 复制正式核验提问

    本提示词只针对当前批次。AI 必须把每家公司放入 verified、rejected 或 unverified 三组之一。

    3. 粘贴正式核验 JSON 并入库

    只有 verifiedStocks 中通过正式准入门的记录会写入正式数据库；rejected 和 unverified 会更新候选池状态。
